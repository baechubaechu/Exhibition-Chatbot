# 전시 구성

이 문서는 졸업전시 **물리 배치**와 **관람 경험 설계**를 정리한다.  
프로젝트 논리는 `00_project_overview.md`, Extra Space 장면은 `02_extra_space.md` §설계 카탈로그를 함께 본다.

## 전시의 핵심 한 가지

**프로젝트:** 환승 인프라 속에서 감정이 전환되는 Extra Space를 만든다.

**전시:** 그 전환의 구조를 관람자가 **직접 읽게** 만드는 번역 장치다.

> X-tra Space는 이동의 공간 안에서 감정의 여백을 만드는 프로젝트이고,  
> 전시는 그 여백이 **어디서, 어떤 건축 요소로, 어떻게** 발생하는지를 감각적으로 해석하게 하는 장치이다.

전시는 작품 설명의 부속물이 아니라, **프로젝트 논리를 한 번 더 수행하는 공간**이다.

## 벽 vs 테이블 역할 분담

| 영역 | 역할 | 관람 거리 |
|------|------|----------|
| **벽** (폭 ~1800) | 이야기·해석·다이어그램·도면·렌더 | 멀리서 한눈에 |
| **테이블** (1800×1200, 관람면 1800) | 물질성·증거·체험 | 가까이서 손·눈 |

벽 = “이 프로젝트가 무엇인가”  
테이블 = “이 공간이 실제로 어떻게 생기고 어떻게 느껴지는가”

## 테이블 구성 (1800 × 1200)

관람면 1800 기준 **좌–중앙–우** 읽기.

| 위치 | 배치 | 비율 가이드 |
|------|------|-----------|
| **중앙** | 전체 모형 | 면적 50~60% |
| **한쪽** | 단면 모형 또는 세부 단면 | 20~25% |
| **앞쪽 가장자리** | 책자 + 방명록 + 1장 요약 시트 | 15~20% |
| **여백** | 필수 | — |

테이블을 꽉 채우지 않는다. 핵심이 안 보인다.

### 테이블 추가 요소 (택 1)

- **재료 샘플 보드** — 감각·물성 (콘크리트·목재·금속·반투명)
- **감정–공간 키워드 카드** — 압도·이완·긴장·침잠·개방 등
- **1장 요약 시트** (A3/A4) — 책자 안 읽는 관람객용

모니터는 테이블 크기에선 복잡해질 수 있어. **QR → 워크스루/설명 페이지**는 가능.

## 벽 구성 (폭 ~1800)

읽는 순서:

1. 메인 렌더 1장 (크게)
2. 개념/전략 다이어그램 2~3
3. 핵심 도면 2~3 (평면·단면)
4. 짧은 설명문

비율 가이드: 이미지 50% / 다이어그램 30% / 도면 20%

### 벽 배치안 (추천)

**안 1 — 상단 이미지 / 하단 정보** (가장 안전)

- 상단 60~65%: 메인 렌더
- 하단: 개념 다이어그램 + 평면/단면 + 사이트/동선

**안 2 — 중앙 축 강조** (감정 흐름·동선 서사가 있을 때)

- 중앙: 메인 렌더 또는 단면 퍼스펙티브
- 좌: 컨셉·사이트
- 우: 평면·단면·세부

이 프로젝트(소멸–재생성–공명, fast/slow flow)에는 **안 1 또는 안 2** 모두 가능.

## 인터랙티브 — Interactive Spatial Index

교수 피드백: 태블릿에서 도면의 공간을 선택 → 모델 해당 위치 조명 → 생성 논리 표시.

### 화면 4항목 (고정)

1. **위치** — 평면에서 선택한 Extra Space
2. **생성 조건** — 어떤 동선 교차 (예: 산책 × 환승 × 보이드)
3. **감각 장치** — 빛·소리·재료·높이·시야·밀도
4. **감정 상태** — 안정·각성·몰입·긴장·회복·해방 등

### 모델 조명 규칙

| 조명 | 의미 |
|------|------|
| 흰색 약한 빛 | 선택된 Extra Space |
| 선형 LED | 연결 동선 |
| 점멸 | 감각 전환 지점 |
| 주변 은은한 빛 | 영향 범위 |

색은 **흰색/웜화이트 + 포인트 컬러 1개**. “예쁜 쇼”가 아니라 정보 계층.

6개 장면과 연동: `02_extra_space.md`의 Threshold~Recovery Edge를 인터랙션 ID로 쓸 수 있다.

## 매체별 역할 (전시장에 다 넣지 않기)

| 매체 | 잘 말해 주는 것 | 이 프로젝트 예 |
|------|----------------|---------------|
| **벽** | 한눈에 읽히는 이미지·다이어그램 | 문제 제기, fast/slow flow, 소멸–재생성–공명 |
| **모형** | 공간 실체·단면·빛 | 전체 + 단면, Extra Space 하이라이트 |
| **책** | 논리·근거·이론·과정 | ANT, 현상학, 크리틱 히스토리 (전시장엔 요약만) |
| **화면/태블릿** | 선택·추적·4항목 해석 | Interactive Spatial Index |
| **QR** | 워크스루·애니메이션 | 테이블 옆 1개 (모니터 대신) |

원칙: 전시장 = **핵심 설득**, 책 = **보관**, 화면 = **체험·변화**, 모형 = **믿음**.

## 벽·테이블에 넣을 우선순위

### 꼭 필요

- 대표 렌더 1
- 전체 이해 다이어그램 1 (예: fast/slow flow, layered field, 소멸–재생성–공명)
- 핵심 전략 다이어그램 1~2
- 평면 1, 단면 1
- 사이트/단절 다이어그램 1 (선택)

### 있으면 좋음

- 구조/채광/동선 다이어그램 1
- 재료·분위기 키워드
- Extra Space 6개 요약 표 (1장)

### 빼도 됨

- 자잘한 도면 여러 장
- 비슷한 렌더 여러 장
- 긴 텍스트

## 자주 묻는 질문

**Q. 챗봇 전시와 물리 전시가 다른가?**  
챗봇은 `canonical` + `sources` 텍스트 기반 해석 장치. 물리 전시는 모형·벽·태블릿. 둘 다 “번역 장치”이지만 매체가 다르다.

**Q. 인터랙션 없이도 되나?**  
된다. 다만 교수 피드백상 **모형 위치 하이라이트 + 4항목 카드/화면**이 프로젝트 논리를 가장 빠르게 전달한다.
