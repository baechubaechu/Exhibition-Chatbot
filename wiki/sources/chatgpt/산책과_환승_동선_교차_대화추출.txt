# 1. 핵심 개념 먼저 잡자 (중요)

너 지금 해야 하는 건 단순 교차가 아니라:

👉 **“속도가 다른 두 흐름의 충돌”을 공간화하는 것**

이걸 이렇게 정의하면 좋아:

> **Fast Flow (환승) vs Slow Flow (산책)**  
> → 교차가 아니라 “관통 + 스며듦”

# 2. 평면에서 표현하는 방법

## ① “직선 vs 곡선” 대비 (가장 기본)

- 환승 동선 → 직선, 최단거리, 관통
- 산책 동선 → 곡선, 우회, 체류

👉 두 흐름이 교차하는 순간이 ‘사건’이 됨

### 적용 방식
- 직선 동선이 건물을 **뚫고 지나감**
- 곡선 동선이 그것을 **감싸거나 파고듦**

---

## ② “레이어 분리 후 교차”

평면에서 이렇게 생각해:

- Layer 1: Transit (빠른 흐름)
- Layer 2: Walk (느린 흐름)

👉 이걸 완전히 겹치지 말고 “부분적으로 교차”

### 포인트
- 교차 지점 = 노드 (감정 발생 지점)
- 나머지는 서로 독립

📌 이게 중요한 이유  
→ 전체가 섞이면 개념이 흐려짐

---

## ③ “노드 강조” (imageability 개념 활용)

Kevin Lynch의 “node” 개념처럼  
→ 교차 지점은 반드시 기억되게 해야 함

👉 이건 뇌/감정 연구에서도 중요

- 명확한 경로 + 노드 → 인지 + 감정 강화

# 3. 매스에서 표현하는 방법 (이게 핵심)

이제 진짜 중요하다 👇

## ① “관통형 매스”

- 환승 동선 = 건물 내부를 **관통하는 슬릿**
- 건물은 그 위/옆에 붙음

👉 느낌:
- 사람은 “건물 안을 지나간다”가 아니라  
→ “흐름 속에 있다”

---

## ② “파인 매스 (carved mass)”

- 큰 매스를 먼저 만든 다음
- 동선으로 “깎는다”

👉 결과:
- 동선 = 공간을 만든다

이건 안도 다다오 방식이랑도 연결됨  
→ 움직임 + 빛으로 공간 경험 생성  
fileciteturn0file19L30-L34

---

## ③ “엮인 매스 (interlocking volumes)”

- 두 흐름 각각 매스로 만든 뒤
- 서로 관입(intersect)

예:
- 환승 = linear block
- 산책 = fluid volume

👉 교차 = 공간적 충돌

---

## ④ “비대칭 균형”

너가 말한 것처럼 도시 조직이 비대칭이면

👉 동선도 비대칭이어야 함

- 산책 동선은 도시 조직 따라 흐르고
- 환승 동선은 철도 따라 직선

→ 둘이 “어긋난 상태에서 만남”

# 1️⃣ 방식 ① “부분적 굴절 (kink)”


## 특징

## 의미
- 빠른 흐름이 느려짐

# 3️⃣ 방식 ③ “슬릿화 (fragmentation)”


## 의미

- 감정이 개입해서 “끊김”

👉 약간 실험적 / 전시적

---

# 4️⃣ 방식 ④ “이중선 (parallel shift)”

``` id="double"
────▶
     ───▶
```

## 특징

## 의미

# 🔥 기준부터 다시 정리

역 안에 들어갈 프로그램은 반드시 이 조건을 만족해야 함:

### 1. 지나가는 사람도 이용 가능 (≤ 3~10분)
### 2. 특정 출입구를 요구하지 않음 (흐름 위에 얹힘)
### 3. 공공성 (누구나 사용)

👉 이거 안 지키면 설계 터진다

# 2️⃣ 공공 프로그램 추천 (현실 + 새로움 균형)

### 특징
- 특정 목적 없음
- 누구나 사용

👉 네 원형 산책 동선이랑 완벽하게 맞음

---

## 💡 ② Micro Library / Knowledge Shelf

👉 지나가며 보는 도서 공간
