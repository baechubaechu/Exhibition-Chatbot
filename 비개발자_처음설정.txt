================================================================================
  전시 챗봇 — 처음 한 번만 하면 되는 설정 (비개발자용)
================================================================================

[중요] 저(AI)는 당신의 Supabase·OpenAI 계정에 로그인할 수 없습니다.
       아래 순서대로 “복사 → 붙여넣기 → 클릭”만 하시면 됩니다.

--------------------------------------------------------------------------------
1) Supabase에서 vector 켜기 + 테이블 만들기
--------------------------------------------------------------------------------
  ① 브라우저에서 supabase.com 로그인 → 새 프로젝트 만들기(또는 기존 프로젝트).
  ② 왼쪽 메뉴 "SQL Editor" 클릭.
  ③ 이 폴더의 파일 "설정용_SQL_한번에_실행.sql" 을 메모장으로 열기 → 전체 복사.
  ④ SQL Editor 빈 칸에 붙여넣기 → 오른쪽 아래 "Run" 버튼 클릭.
  ⑤ 에러 없이 끝나면 성공. (이미 있으면 "already exists" 비슷한 메시지는 무시해도 됨)

--------------------------------------------------------------------------------
2) Supabase에서 복사해 둘 값 (나중에 .env.local 에 붙임)
--------------------------------------------------------------------------------
  ① 왼쪽 "Project Settings" (톱니바퀴) → "API".
  ② Project URL  →  SUPABASE_URL 로 쓰임
  ③ service_role 의 secret 키 (절대 유출 금지) → SUPABASE_SERVICE_ROLE_KEY

--------------------------------------------------------------------------------
3) OpenAI 키
--------------------------------------------------------------------------------
  ① platform.openai.com 로그인 → API keys → Create new secret key
  ② 복사한 값 → OPENAI_API_KEY

--------------------------------------------------------------------------------
4) .env.local 파일
--------------------------------------------------------------------------------
  ① 이 폴더(exhibition-chatbot) 안에 ".env.local" 이라는 이름의 파일이 있으면 메모장으로 연다.
     없으면 새로 만든다. (앞에 점이 있어야 함)
  ② 같은 폴더의 ".env.local.채워넣기.txt" 를 열어서 안내 대로
     SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, OPENAI_API_KEY, ADMIN_SECRET
     네 군데만 본인 값으로 바꾼 뒤 전체를 .env.local 에 저장.

  ADMIN_SECRET = 전시 관리자 페이지 비밀번호로 쓸 아무 긴 영문숫자(본인만 아는 것).

--------------------------------------------------------------------------------
5) Node.js 설치 (한 번만)
--------------------------------------------------------------------------------
  nodejs.org 에서 LTS 버전 설치 → 설치 후 PC 재시작 한 번 권장.

--------------------------------------------------------------------------------
6) 터미널에서 한 번만: npm install
--------------------------------------------------------------------------------
  exhibition-chatbot 폴더에서:
    npm install

--------------------------------------------------------------------------------
7) 원문 넣은 뒤 — 내가 할 일 (둘 중 하나)
--------------------------------------------------------------------------------
  A) "소화(자동 초안)까지" 돌리기 — 시간·API 비용 많음
     → "2_원문소화후_벡터올리기.bat" 더블클릭
     → 끝나면 wiki/canonical/*.md 맨 아래에 초안이 붙음 → 꼭 읽고 고친 다음 다시 bat 실행 권장

  B) "벡터만 갱신" (원문 폴더만 바꿨을 때, 소화 안 함)
     → "1_벡터만_올리기.bat" 더블클릭

--------------------------------------------------------------------------------
8) 챗봇 실행
--------------------------------------------------------------------------------
  터미널에서:  npm run dev
  브라우저:    http://localhost:3000

--------------------------------------------------------------------------------
9) Cursor 안에서 나(AI)에게 시키고 싶을 때
--------------------------------------------------------------------------------
  "exhibition-chatbot 폴더에서 npm run sync:knowledge:with-digest 실행해줘"
  또는 "원문 넣었으니 ingest만 해줘" 처럼 말하면 됩니다.
